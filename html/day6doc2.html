Style Guide

There are some practices that can be followed to make our code clean

  • using lowercase attribute names
  • closing html elements
  • declaring doctype
  • quote attribute values
  • improve readability by indenting and adding blank lines when necessary
  • include lang, viewport settings, and charset
  • use stylesheets to improve presentation

HTML Entities

HTML entities or escape characters are used to display reserved characters. For example the less than and greater than symbols are used to open and close tags in HTML and therefore have a special meaning assigned if we type this symbol as regular text it will not be recognized as such. Therefore we have entities of the format '&entity;'. Each such speacial character has both an entity name and an entity number.

XHTML

The eXtensible HyperText Markup Language is an XML based version of HTML. It follows a stricter syntax than HTML. There are certain ways of coding that are accepted by HTML but not XHTML. For example unqouted lowercase attribute names are mandatory in XHTML but not in HTML.
